How it works · accuracy

Audio is transcribed in ~28-second windows by a 0.9B model (Whisper-Medium encoder + Qwen3-0.6B decoder), served as an F16 GGUF. Whisper-style 80-bin log-mel features are computed in JavaScript; the encoder and decoder run in WebAssembly through RapidSpeech.cpp (llama.cpp/ggml lineage), multi-threaded via SharedArrayBuffer under cross-origin isolation.

Speaker labels ([S01], [S02], …) are produced directly by the model as part of its output — there is no separate ECAPA voice-fingerprint or cross-window clustering pass here. Speaker identity is therefore assigned per the model's own tags within each decoding window; the same S-number in two far-apart windows is not guaranteed to be the same physical voice.

Everything (model, audio, results) stays in this tab. Speed depends on your hardware and thread count. The transcript streams window by window and you can stop anytime, keeping partial results.
